05 — sql analysis

Fraud logic in SQL

Key queries calculating fraud rates, segmented risk groups, and surfacing patterns by merchant, origin, and velocity.

fraud_detection_query.sql 
-- Overall fraud rate
SELECT
  COUNT(*) AS total_transactions,
  SUM(CASE WHEN is_fraud = 1 THEN 1 ELSE 0 END) AS fraud_count,
  ROUND(AVG(is_fraud::numeric) * 100, 2) AS fraud_rate_pct
FROM credit_card_transactions;

-- Fraud rate by merchant category
SELECT
  merchant_category,
  COUNT(*)                                             AS total,
  SUM(is_fraud)                                        AS fraud_count,
  ROUND(SUM(is_fraud)::numeric / COUNT(*) * 100, 2) AS fraud_rate,
  SUM(amount * is_fraud)                               AS fraud_amount
FROM credit_card_transactions
GROUP BY merchant_category
ORDER BY fraud_amount DESC;

-- High-velocity fraud detection
WITH velocity AS (
  SELECT
    customer_id, transaction_date,
    COUNT(*) OVER (
      PARTITION BY customer_id
      ORDER BY transaction_date
      RANGE BETWEEN INTERVAL '1 day' PRECEDING AND CURRENT ROW
    ) AS txn_last_24h, is_fraud
  FROM credit_card_transactions
)
SELECT
  CASE WHEN txn_last_24h >= 5 THEN 'High Velocity'
       WHEN txn_last_24h >= 3 THEN 'Medium Velocity'
       ELSE                        'Normal'
  END AS velocity_group,
  ROUND(AVG(is_fraud::numeric)*100,2) AS fraud_rate_pct
FROM velocity
GROUP BY 1
ORDER BY fraud_rate_pct DESC;
06 — business recommendations

Six controls to deploy
right now

Each recommendation is tied directly to a SQL finding — prioritized by fraud impact and implementation urgency.

REC 01 — Foreign Monitoring
🌐
Strengthen verification for foreign transactions
Cross-border transactions carry a 4.08% fraud rate versus just 0.70% domestically — a 3.2× gap. Implement a dedicated verification layer (step-up authentication, SMS OTP, or callback confirmation) triggered automatically on any foreign card activity.
Supporting query: Q6 · Foreign txns: 4.08% fraud rate vs 0.70% domestic
High Priority
REC 02 — Velocity Caps
⚡
Implement dynamic rate-limiting at 5+ txns / 24h
High-velocity customers represent only ~12% of total transaction volume but account for ~38% of all fraud cases — a 4.75% fraud rate versus 0.93% in normal-velocity groups. Enforce a soft cap at 5 transactions per 24 hours with escalating friction above that threshold.
Supporting query: Q11 · High velocity: 4.75% fraud rate, 38% of all fraud
High Priority
REC 03 — Device Trust
📱
Auto OTP / biometric for low device trust scores
Devices scoring below 50 on the trust scale drive 4.24% fraud rates and contribute ~59% of all fraud despite covering the smallest volume segment. Any transaction on an unrecognized or low-trust device should auto-trigger OTP or biometric re-verification before approval.
Supporting query: Q10 · Low trust bin: 4.24% rate, 59% of all fraud cases
High Priority
REC 04 — Combined Signal
🔗
Flag foreign + location mismatch combinations immediately
When a foreign transaction also carries a location mismatch, fraud rate spikes to 6.2% — over 10× the dataset baseline. This specific combination should trigger an immediate hold and manual review or automated block, rather than standard friction.
Supporting query: Q8 · Foreign + mismatch combined: 6.2% fraud rate (10× baseline)
Immediate Action
REC 05 — Category Scoring
🏪
Apply per-category fraud score thresholds
Travel (2.21%) and Grocery (2.06%) carry the highest fraud rates by category. Because transaction amounts in these categories are large, financial impact per fraud event is disproportionate. Apply tighter approval thresholds and enhanced monitoring for these two categories specifically.
Supporting queries: Q2, Q12 · Travel & Grocery: highest fraud rate and total fraud amount
Medium Priority
REC 06 — ML Uplift
🧠
Introduce real-time ML risk scoring to go beyond rules
The rule-based system combining all three controls (foreign flag + velocity cap + device trust) catches an estimated ~60% of fraud. A real-time logistic regression or gradient-boosted model trained on all engineered features (ROC-AUC ~0.89) would significantly close that gap and reduce false positives simultaneously.
Model evaluation · Logistic regression: Precision 0.74 · Recall 0.68 · ROC-AUC 0.89
Strategic
